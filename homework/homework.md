# Домашнее задание: Закрепление материала по Telegram-ботам на Go

## Задание 1: Добавление новой команды /info

**Описание:**  
Добавьте в бота новую команду `/info`, которая будет выводить информацию о пользователе, отправившем команду. Бот должен отправить сообщение с данными пользователя: его ID, имя пользователя (если есть), имя и фамилию.

**Ожидаемый результат:**  
При отправке команды `/info` бот отвечает сообщением вида:
```
Информация о вас:
ID: 123456789
Имя пользователя: @username
Имя: Иван
Фамилия: Иванов
```

Если какое-то поле отсутствует (например, username), бот должен указать "не указано".

---

## Задание 2: Обработка текстовых сообщений с числами

**Описание:**  
Модифицируйте обработку обычных текстовых сообщений так, чтобы бот определял, является ли сообщение числом. Если пользователь отправил число, бот должен вычислить и отправить квадрат этого числа. Если сообщение не является числом, бот должен просто повторить текст пользователя, как сейчас.

**Ожидаемый результат:**  
- При отправке "25" бот отвечает: "Квадрат числа 25 равен 625"
- При отправке "привет" бот отвечает: "Вы написали: привет"
- При отправке "3.14" бот отвечает: "Квадрат числа 3.14 равен 9.8596"

## Задание 3: Команда /settings с проверкой администратора

**Описание:**  
Создайте команду `/settings`, которая будет доступна только администраторам бота. Если команду отправляет администратор (его ID есть в списке `ADMIN_IDS` из конфигурации), бот должен отправить сообщение с текущими настройками бота (режим отладки, таймаут). Если команду отправляет обычный пользователь, бот должен ответить сообщением о том, что эта команда доступна только администраторам.

**Ожидаемый результат:**  
- Администратор получает: "Настройки бота:\nРежим отладки: true\nТаймаут: 60 секунд"
- Обычный пользователь получает: "Эта команда доступна только администраторам"

## Задание 4: Обработка команды /echo с аргументами

**Описание:**  
Добавьте команду `/echo`, которая принимает аргументы после команды. Бот должен повторить все аргументы, которые пользователь указал после команды. Если аргументы не указаны, бот должен попросить пользователя указать текст для повторения.

**Ожидаемый результат:**  
- При отправке `/echo привет мир` бот отвечает: "привет мир"
- При отправке `/echo` бот отвечает: "Пожалуйста, укажите текст для повторения. Использование: /echo <текст>"
- При отправке `/echo это тест 123` бот отвечает: "это тест 123"

## Задание 5: Улучшение команды /help

**Описание:**  
Модифицируйте команду `/help` так, чтобы она выводила список всех доступных команд с их кратким описанием. Список должен включать все команды, которые реализованы в боте: `/start`, `/help`, `/info`, `/settings`, `/echo`, а также информацию о текстовых командах (например, "подписка").

**Ожидаемый результат:**  
При отправке `/help` бот отвечает сообщением:
```
Список доступных команд:

/start - начать работу с ботом
/help - показать это сообщение
/info - получить информацию о себе
/settings - настройки бота (только для администраторов)
/echo <текст> - повторить указанный текст

Текстовые команды:
подписка - информация о подписке
```
